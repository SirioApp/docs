# Roles and Responsibilities

Backed has three primary actor classes.

## Founders

Founders are responsible for the project they create and the correctness of the configuration they submit.

Founders should:

- confirm the correct agent ID before creation;
- verify timing, token metadata, and collateral configuration;
- avoid public distribution before operator approval;
- verify they are acting in the intended environment.

## Investors

Investors commit collateral into sale contracts and rely on onchain sale state for claims, refunds, and redemption paths.

Investors should:

- verify whether they are on testnet or mainnet;
- verify the project identifier and sale timing;
- confirm the collateral token is USDM;
- treat contract state as authoritative if interface output is unclear.

## Operators

Operators own the highest-risk workflows in the platform.

Their responsibilities include:

- approving projects;
- verifying environment-specific contract addresses;
- handling privileged write operations;
- maintaining treasury execution boundaries;
- responding to mismatches between UI output and onchain truth.

## Responsibility boundaries

| Decision | Owner |
| --- | --- |
| Agent identity creation | Founder or identity workflow owner |
| Raise creation | Founder or delegated operator |
| Raise approval | Authorized operator |
| Sale participation | Investor |
| Treasury execution permissions | Authorized operator |
| Source of truth during disputes | Deployed contracts |
