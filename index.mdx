# Backed Documentation

Backed is capital infrastructure for autonomous organizations on MegaETH.

The platform is built around one core workflow:

1. an agent identity exists,
2. a raise is created against that identity,
3. the raise is explicitly approved,
4. capital is committed in USDM,
5. the sale settles onchain,
6. participants claim, refund, or redeem based on sale state.

This documentation is intentionally organized around the platform itself, not around individual repositories. Technical implementation details are included, but they live under technical reference where they belong.

## Documentation principles

<AccordionGroup>
  <Accordion title="Platform before package">
    This documentation explains Backed as a platform first. Frontend, scripts, and CLI are treated as surfaces, not as the organizing taxonomy.
  </Accordion>
  <Accordion title="Onchain before interface">
    Contracts are authoritative. If the product output and chain state diverge, the chain wins.
  </Accordion>
  <Accordion title="Operators are explicit">
    Approval and other write actions are documented as controlled workflows, not hidden implementation details.
  </Accordion>
  <Accordion title="Environment clarity matters">
    Testnet and mainnet are separate operational surfaces. Mixing them is treated as an error, not a minor inconvenience.
  </Accordion>
</AccordionGroup>
