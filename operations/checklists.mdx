# Operational Checklists

Use these checklists for the workflows most likely to create operational mistakes.

## Before creating a raise

- Confirm the intended environment.
- Confirm the intended agent ID.
- Confirm collateral assumptions.
- Confirm launch delay and duration.
- Confirm token name and symbol.

## Before approving a project

- Confirm the project ID.
- Confirm the project exists on the intended environment.
- Confirm the signer has the right role.
- Confirm the project is the one intended for launch.

## Before treating a raise as live

- Confirm approval is complete.
- Confirm the intended environment is the one users will access.
- Confirm the sale window is correct.
- Confirm public-facing links reference the correct project.

## Before troubleshooting a discrepancy

- Confirm testnet versus mainnet.
- Confirm the relevant contract addresses.
- Confirm the project ID.
- Read current state directly from the contracts.
